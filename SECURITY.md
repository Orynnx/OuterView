# 安全策略

## 报告漏洞

请不要在公开 Issue 中附带设备序列号、账号信息、完整 `/data` 文件、签名密钥或包含个人通知内容的日志。公开仓库启用 Security Advisories 后，请优先通过私有漏洞报告；否则只提交不含利用细节的 Issue，请维护者建立私密沟通渠道。

## 信任模型

OuterView 是高权限 LSPosed 模块，会在系统背屏宿主进程中执行。只从可信 Release 安装，并核对签名。卡片 MAML 也可能包含 Intent 或外部命令，风险确认不是沙箱，请只导入可信 ZIP。

## 明确不做

- 不修改系统 `notification_widget.json`。
- 不删除系统卡片模板。
- 不接受卡片包指定宿主路径或最终 business。
- 不在仓库中保存生产签名密钥。
